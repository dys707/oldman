# 基于 RAG 架构 + 三层长期记忆的老年人心理健康对话系统

本项目基于 RAG 检索增强生成架构，结合本地 Embedding 与大模型，实现专业知识精准问答；
通过**短期记忆 / 里程碑记忆 / 语义记忆**三层记忆结构，突破传统对话框限制，让模型具备真正的长期记忆能力。

---

## 核心功能

1. **RAG 架构精准检索**
   - 基于专业知识库进行检索
   - 结合大模型生成准确、可靠的回答

2. **本地模型部署**
   - 本地配置 Embedding 模型
   - 本地调用 LLM 进行分析与生成

3. **三层长期记忆系统（核心亮点）**
   - **短期记忆**：记录实时对话内容，自动向量化存入数据库
   - **里程碑记忆**：基于短期记忆筛选与缓存，记录用户阶段性状态
   - **语义记忆**：基于短期记忆筛选与缓存，构建用户性格、三观、重大背景等人物画像

---

## API 接口

项目最终封装为 5 个标准化 API：

- `/health`
  健康检查接口

- `/api/rag/answer`
  对话问答主接口

- `/api/rag/short_term_memories`
  返回短期记忆（可用于前端可视化）

- `/api/rag/milestone_term_memories`
  返回里程碑记忆

- `/api/rag/semantic_term_memories`
  返回语义记忆

## 使用须知

1. **模型配置**
   - 需自行创建 `models` 目录
   - 默认使用：`./models/text2vec-base-chinese`
   - 更换模型时，代码路径需保持一致

2. **知识库资料准备**
   - 在根目录创建：`data/raw`
   - 放入项目所需专业资料

3. **向量库构建**
   运行以下脚本生成向量数据库：
   - `temp1`：资料文件分割
   - `temp2`：资料文件向量化
   - 生成后存储于：`vector_db/chroma_store`

4. **角色提示词说明**
   项目初衷为**老年人心理健康服务**，因此：
   - `memory_manager.py`
   - `qa_interface.py`
   中所有 LLM Prompt 默认设置为：
   **“你是一个专业的老年人心理助手”**
   可直接修改，不影响记忆机制。
